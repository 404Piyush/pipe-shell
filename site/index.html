A POSIX-ish command interpreter in C11.

Pipelines. Redirection. Backgrounding. Built-ins. A recursive parser, the full pipeline recipe, and an interactive REPL. ~500 LOC. 56 assertions, all passing.

Install View API 
$ git clone https://github.com/404Piyush/pipe-shell.git
$ cd pipe-shell && make test
56 passed, 0 failed
$ ./pipe-shell
pipe-shell$ ls /etc | grep hosts | wc -l
       3
pipe-shell$ exit

What's in the box

Recursive parser

Line in, AST out. Splits on |, then tokenizes each stage on whitespace, honouring <, >, and >> as metacharacters.

General pipeline recipe

N-1 pipes, N children, N-1 parent-side close()s. The executor is 30 lines. Tested with 1, 2, and 3 stages plus redirection in every position.

Interactive & one-shot

Run it as a REPL (./pipe-shell) or one-shot (./pipe-shell --run "cmd | cmd"). The --show flag prints the AST before running.

See it in action

$ ./pipe-shell --run "ls /etc | grep hosts | wc -l"
       3

$ ./pipe-shell --run "echo hello world > /tmp/out.txt; cat /tmp/out.txt"
hello world

$ ./pipe-shell --run "wc -l < Makefile"
      25

$ ./pipe-shell --run "ls /usr | grep stdio --show"
stage[0]: [ls] [/usr]
stage[1]: [grep] [stdio]  > /dev/null
(no output on stdout; printed to /dev/null)

API at a glance

The library surface is three functions:

bool shell_parse    (const char *line, shell_cmd *out);
int  shell_run      (const shell_cmd *cmd);
void shell_cmd_print(const shell_cmd *cmd, FILE *f);

The four rules of pipelines

  1. Parent closes its copy of every pipe fd (or consumer hangs forever).
  2. Child dup2s the read end to stdin, the write end to stdout.
  3. Child closes all fds it doesn't need before exec.
  4. Parent waitpids every child (foreground) or none (background &).

Limitations (intentional)

  • No glob (*) or variable ($HOME) expansion.
  • No quoting of any kind.
  • No ;, &&, ||. A line is one pipeline.
  • No job control. Background jobs are fire-and-forget.

Each missing feature is a 100-200 line change on top of the existing structure — a good follow-up project.
